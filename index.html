🛡️ Middleware Chain
Applied in order on POST /api/proposals (new quote creation) — strictest request path in the app
1
requireAuth
Validate Bearer JWT via Supabase. Compare session_id against users.active_session_jti. Mismatch → 401 "logged in elsewhere".
all protected routes
↓
2
requireSubscription
Block if subscription_status is expired / cancelled / past_due. Returns 402 Payment Required.
new proposals only
↓
3
requirePaid
Block churned / rejected / pending_review testers. 30-second result cache. Admin email bypasses.
anti-abuse
↓
4
checkProposalLimit
Monthly quota: warn at 15 proposals, hard block at 20. Attaches req.proposalUsage for frontend banner.
rate limit
↓
5
errorHandler
Global — mounted last in index.js. Logs full error server-side, returns friendly { success: false, error } to client.

📐 Key Backend Rules
Response format
{ success: true, data: {} }
{ success: false, error: "…" }
Paystack amounts
Always multiply ZAR by 100. R10 = 1000. R299 = 29900. Never send floats.
AI model
claude-haiku-4-5 only. Never change without asking Craig. Rate limit: 5/hour/user.
Email provider
Resend only. Never SendGrid. From address set via RESEND_FROM_EMAIL + RESEND_FROM_NAME env vars.
Supabase client
Backend uses service key — bypasses RLS. JWT auth validated via supabase.auth.getUser(token).
Reference numbers
Generated via atomic Supabase RPC. Format: {prefix}-001, revisions: {prefix}-001-R1.

📐 Schema Change Rules
Adding columns
ALTER TABLE ... ADD COLUMN IF NOT EXISTS
Never edit original CREATE TABLE block
Views after column add
DROP VIEW first, then CREATE VIEW. CREATE OR REPLACE fails when new columns are involved.
RLS on new tables
Always click "Run and enable RLS" in Supabase UI. Backend service key bypasses RLS regardless.
SQL Editor tip
Always open a new tab (+) in the SQL editor. Don't reuse old tabs. Prior DDL already committed on partial runs.
Never in production
No DROP TABLE or DROP COLUMN without a backup. Migrations live at bottom of schema.sql under -- MIGRATIONS.
jsonb conventions
rate_card: {deposit_percent, quote_validity_days}. scope_template: [{id,name,unit_price,unit}]. line_items: [{description,unit_price,unit,quantity,amount}].
